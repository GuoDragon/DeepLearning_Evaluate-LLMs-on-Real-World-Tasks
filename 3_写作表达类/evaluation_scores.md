## 三、评分标准

每个维度采用 1-5 分制，单条输出满分 25 分。第五项“克制度”用于评价是否避免过度夸张；分数越高代表越克制、越少虚构。

| 分数 | 判定标准 |
|---:|---|
| 5 | 表现优秀：表达自然，结构清楚，准确保留原意，完全适配目标场景，基本无夸张或虚构。 |
| 4 | 表现良好：整体可直接使用，仅有轻微冗余、轻微扩展或个别措辞不够精准。 |
| 3 | 基本可用：主体方向正确，但存在较明显新增信息、场景偏差、套话或营销化表达，需要人工修改。 |
| 2 | 问题较多：虽然有部分可用内容，但偏离原意、虚构细节、过度包装或格式不合适较明显。 |
| 1 | 不建议使用：严重偏题、明显编造、夸张承诺或无法满足目标场景。 |

| 指标 | 评价重点 |
|---|---|
| 语言流畅性 | 语法、用词、句式是否自然通顺，中英文是否符合常用表达。 |
| 逻辑结构 | 是否条理清楚、层级合理、重点突出，是否便于直接使用。 |
| 表达准确性 | 是否保留原始含义，是否避免新增未经提供的信息、数据、技术栈、年份或结论。 |
| 场景符合度 | 是否符合职场、校园、简历、学术、电商、月报、科普等目标语境。 |
| 克制度 | 是否避免“最强”“极致”“百分百”“旗舰级”等过度夸张，是否避免不实承诺。 |

## 六、结果分析

### 6.1 总体结论

- 从内容表现看，`Output-C`整体最稳定，平均总分最高，尤其在改进 Prompt 下几乎都能做到“直接给出可用结果”，较少加入未经提供的事实。
- `Output-D`和`Output-E`在多数样例中也表现较好，但在产品文案、简历项目和社团招新这类容易发挥的场景中，偶尔会增加未经确认的技术、年份、宣传细节。
- `Output-A`的结构化能力较强，改进 Prompt 后英文摘要和科普写作提升明显，但部分中文职场/产品输出存在新增具体信息的问题。
- `Output-B`语言感染力强，但也是最容易“过度包装”的一组，尤其在社团招新、简历项目和产品说明中，新增事实和营销化表达较多。
- 五组输出共同的短板不是语言通顺度，而是“表达准确性”和“克制度”：模型很容易为了让文本更好看而补充原文没有提供的信息。

### 6.2 Prompt 改进效果

- 改进 Prompt 的平均总分为 `22.80/25`，高于基础 Prompt 的 `21.58/25`，说明加入角色、格式、约束条件后，整体输出质量有提升。
- 提升最明显的维度是逻辑结构和表达准确性。逻辑结构从 `4.43` 提升到 `4.82`，表达准确性从 `3.78` 提升到 `4.16`。
- 改进 Prompt 对社团招新、产品卖点、工作总结这类需要结构化输出的任务帮助较大，但不能完全消除模型主动补充细节的问题。
- 英文类任务整体更稳定，尤其样例2、6、9在多数输出中语法和语气都较好；中文营销类和简历类任务更容易出现夸张或虚构。

### 6.3 典型优势

- 语言流畅性普遍较高：绝大多数输出能修正口语化表达，使文本更正式、自然。
- 结构化能力较强：改进 Prompt 下，多数答案能够按照申请、通知、卖点、工作总结等格式组织内容。
- 场景适配总体可靠：请假申请、英文消息、英文课程论文、科普短文等场景表现较稳定。
- 简洁任务效果好：样例2、样例3、样例6、样例10通常能得到较高分，因为任务边界清楚，不需要大量发挥。

### 6.4 主要问题与失败模式

- 新增未经提供的信息：简历场景中常见新增 Tkinter、PyQt、MySQL、数据持久化、Top X%；月报场景中常见新增报表数量、数据准确率、流程优化方案。
- 过度营销化：产品文案中常见“旗舰级品质”“极致性价比”“360°倒置不漏”“先进真空工艺”等未提供承诺。
- 过度创意化：社团招新场景中常见虚构社团名、年份、名额有限、赛事级别、社长寄语等，虽然吸引人，但准确性下降。
- 输出不够直接：基础 Prompt 下部分答案给出多个版本或附带解释，实际使用时需要人工筛选。
- 英文细节问题：个别答案存在不自然搭配或语法错误，例如“This paper objective examines”等，需要人工校对。

### 6.5 后续报告可引用的结论

- 对写作表达类任务而言，LLM 最擅长提升语言流畅性和文本结构，尤其适合将口语化文本改写为正式文本。
- LLM 的主要风险是为了增强表达效果而主动补充细节，因此在简历、商务产品、工作总结等场景中必须人工核查事实。
- 改进 Prompt 可以显著改善格式和场景适配，但仍需要明确加入“不得新增事实、不得虚构参数、不得夸大宣传”等约束。
- 如果用于真实场景，建议采用“LLM 初稿 + 人工事实核查 + 人工删减夸张表达”的流程，而不是直接复制最终输出。
