````markdown
同学你好！这个问题是深度学习的“第一道坎”，搞懂它，后面很多问题就迎刃而解了。我用最直白的方式给你讲清楚。

---

### 1. 核心定义（一句话版）

- **过拟合**：模型**死记硬背**了训练数据，连“噪音”和“偶然”都记下来了，导致它学得太“专”，换一套新题目就不会做了。
- **欠拟合**：模型**没学到位**，连训练数据里的基本规律都没抓住，像个懵懂的新手，做什么题都错。

---

### 2. 直观生活类比（考试复习）

想象你下周要考“动物识别”这门课，复习资料是**100张动物图片**（训练集）。

- **过拟合**（死记硬背的学霸）：  
  他把每张图片的像素点都背下来了。考试时，只要图片和复习资料一模一样，他满分；但只要图片换了个角度（比如猫从侧面变成正面），他就懵了，因为他背的是“像素”，不是“猫的特征”。

- **欠拟合**（摆烂的学渣）：  
  他复习时只扫了一眼，总结出“所有动物都是四条腿”。考试时，不管猫、狗、还是大象，他全答“四条腿动物”，连基本的毛色、体型都分不清。结果复习题和考试题，他都是一塌糊涂。

---

### 3. 在训练集和测试集上的典型表现（关键区别）

| 指标           | **过拟合**                               | **欠拟合**                           |
| -------------- | ---------------------------------------- | ------------------------------------ |
| **训练集表现** | **非常好**（准确率极高，损失几乎为0）    | **很差**（准确率低，损失高）         |
| **测试集表现** | **很差**（和训练集差距巨大，准确率暴跌） | **也很差**（和训练集差不多，都很低） |

> **核心区别就看一句话**：  
> 过拟合是“训练好、测试差”（差距大）；欠拟合是“训练差、测试也差”（都差，差距不大）。

---

### 4. 常见解决方法（各给两种，术语会解释）

#### 针对过拟合（学太“过”了）：
- **增大训练数据**：多找些不同角度、不同光照的动物图片。数据多了，模型就知道“像素不重要，特征才重要”。
- **正则化（给模型戴“紧箍咒”）**：比如L2正则化，简单说就是在损失函数里加一项“模型不能太复杂”的惩罚，逼模型把权重变小，让它不敢死记硬背。
- **早停法**：训练过程中，一旦发现测试集效果不再上升甚至下降，就立刻停止训练，防止继续“背”下去。

#### 针对欠拟合（学“不够”）：
- **增加模型复杂度**：比如把神经网络层数加多、每层神经元加多，让模型有更强的“脑容量”去抓规律。
- **训练更久（增加迭代次数）**：让模型多“看”几遍训练数据，给它足够时间学习。
- **特征工程（人工提点）**：手动添加一些明显有用的信息（比如把“动物体重”也作为输入），帮助模型降低学习难度。

---

### 最后一句总结（记好这个区别）：
> **欠拟合**是“能力不够”，**过拟合**是“记性太好，但没理解”。  
> 我们的目标，是在两者之间找一个**刚刚好**的平衡点。

如果有哪个点还不清楚，随时追问，我用更土的例子给你讲 😄
````

## 样例2

````markdown
好的，同学。这个问题非常核心，很多初学者一开始都会把这两个词搞混。作为助教，我帮你彻底理清楚。

---

### 1. 一句话概括二者各自的作用
- **梯度下降**：是一个**“优化算法”**，作用是用计算出的梯度来**更新模型的参数**，让损失变小。
- **反向传播**：是一个**“计算方法”**，作用是高效地**计算损失函数关于每个参数的梯度**（即偏导数）。

---

### 2. 分别解释基本流程

**🔽 反向传播（Backpropagation）—— 从后往前算梯度**

1. 前向传播：输入数据经过每一层网络，算出最终预测值。
2. 计算损失：用损失函数比较预测值和真实标签，得到一个损失值（标量）。
3. 链式求导：从输出层开始，利用微积分中的链式法则，逐层向前（往输入方向）计算损失对每个权重和偏置的偏导数（梯度）。
4. 结果：每一层的参数都得到对应的梯度值（一个向量或张量）。

> 记住：反向传播**只算数，不更新参数**。

**⬇️ 梯度下降（Gradient Descent）—— 用梯度去更新参数**

1. 拿到反向传播算好的所有参数的梯度。
2. 对每个参数 `θ`，按照公式更新：  
   `θ_new = θ_old - 学习率 × 梯度`
3. 重复这个过程，直到损失收敛。

> 记住：梯度下降**只更新，不算梯度**（它依赖别人给它梯度）。

---

### 3. 二者在一次神经网络训练迭代中的先后关系

一次迭代（Iteration）的顺序是：

1. **前向传播**（算预测值）  
2. **计算损失**  
3. **反向传播**（计算每个参数的梯度） ← 先发生  
4. **梯度下降**（用这些梯度更新参数） ← 后发生  

**结论**：反向传播是“侦察兵”，先探路算出坡度；梯度下降是“施工队”，后根据坡度调整位置。**先反向传播，后梯度下降**，二者缺一不可，但次序固定。

---

### 4. 一个简单比喻

想象你在一个完全黑暗的山上，想走到山脚。

- **反向传播（计算梯度）** = 你用脚往各个方向踩一踩，感觉哪边更陡、坡度多大。这一步只获取“地面信息”，但你人还没动。
- **梯度下降（更新参数）** = 根据刚才感知到的坡度和方向，你实际迈出一步（步长 = 学习率），移动到新的位置。

**区别**：  
“计算梯度”是**感知**（只测量，不移动），  
“更新参数”是**行动**（只移动，不测量）。

---

### 5. 初学者常见误区

| 误区                               | 正确理解                                                                                         |
| ---------------------------------- | ------------------------------------------------------------------------------------------------ |
| “反向传播就是梯度下降”             | ❌ 反向传播是**求梯度的方法**，梯度下降是**用梯度更新的方法**，完全两回事。                       |
| “没有反向传播也能做梯度下降”       | ❌ 没有梯度，梯度下降根本不知道往哪更新。但可以用数值微分（很慢）代替，只是实际都用反向传播。     |
| “梯度下降只在反向传播之后运行一次” | ✅ 对，每次迭代只更新一次。但反向传播会遍历所有层。                                               |
| “反向传播会改变参数”               | ❌ 反向传播**绝不改变**任何参数，只改变**梯度值**（存储在变量中）。参数改变只发生在梯度下降步骤。 |
| “学习率影响反向传播”               | ❌ 学习率只影响梯度下降的更新步长，不影响反向传播算出的梯度数值。                                 |
| “梯度下降只用在神经网络”           | ❌ 梯度下降是通用优化算法，线性回归、逻辑回归都用；但反向传播**专用于多层神经网络**。             |

---

如果还有疑问，记住这句口诀：  
**反向传播算坡度，梯度下降迈步子；先算后走，次序别乱。**
````

## 样例3

````markdown
1. **核心结论**  
Batch Normalization（BN）让网络每一层的输入分布保持相对稳定，**本质上是给深层网络加了一个“动态数据缩放器”**。它的最大好处不是玄学的“分布对齐”，而是**让梯度更可靠、允许更大学习率、让训练对参数初始化不敏感**，从而显著加速收敛。

2. **Batch Normalization 在每个 mini-batch 中做了什么**  
   对一个 mini-batch 的某一层输入（比如 \(B = \{x_1, ..., x_m\}\)），BN 做三步：  
   - **计算均值和方差**：\(\mu_B = \frac{1}{m}\sum x_i\)，\(\sigma_B^2 = \frac{1}{m}\sum (x_i-\mu_B)^2\)。  
   - **标准化**：\(\hat{x}_i = \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}\)，让这批数据变成均值为 0、方差为 1。  
   - **缩放和平移**：\(y_i = \gamma \hat{x}_i + \beta\)，其中 \(\gamma, \beta\) 是可学习参数，让网络可以“反悔”标准化，保留必要的表达能力。

3. **它如何影响梯度传播、学习率选择和训练稳定性**  
   - **梯度传播**：标准化把每层输入压到了梯度较大的“非饱和区”（比如 Sigmoid 的线性区），避免深层梯度消失。同时，反向传播时 BN 会除以标准差，这相当于对梯度做了**自动白化**，使得不同维度的梯度量级相近，不会出现某些参数梯度极大、某些极小。  
   - **学习率选择**：因为梯度尺度被控制住，不会因为某层输入爆炸导致梯度爆炸，所以可以用更大的学习率（通常能大 5~10 倍）而不会发散。  
   - **训练稳定性**：参数更新会改变前一层输出，但 BN 会在当前 mini-batch 内重新居中缩放，相当于**吸收了前层参数的随机波动**，让后层看到的输入变化更平滑，训练曲线更稳。

4. **训练阶段和推理阶段的差异**  
   - **训练时**：均值和方差由当前 mini-batch 计算，所以同一层的统计量随 batch 不同而波动——这反而带来轻微的**正则化效果**（类似 Dropout）。  
   - **推理时**：不再依赖 batch，而是使用**训练集上所有 mini-batch 统计量的移动平均**（固定 \(\mu_{running}, \sigma_{running}\)），并且 \(\gamma, \beta\) 固定。推理时 BN 是一个确定性线性变换，不会因为单样本改变。

5. **一个常见误区或使用注意事项**  
   **常见误区**：认为 BN 可以随便加在任意位置，且 batch size 越小越好。  
   **真相**：BN 强烈依赖 batch 内统计量的代表性。**当 batch size 太小（如 ≤8）时，均值和方差噪声很大**，反而会损害训练。此时应改用 Layer Norm 或 Group Norm。另外，BN 不适合 RNN 或变长序列，因为不同时间步的统计特性不同。使用时要确保训练和推理的 \(\epsilon\) 一致，且移动平均更新时要“去偏”（有的框架默认已做）。
````

````markdown
- **图像分类**：**优先 CNN**。因为图像具有天然的局部空间平移不变性，CNN 的卷积和池化操作能高效利用这一先验，参数量少且训练稳定，在小到中型数据集上远优于 Transformer（ViT 需要海量数据或蒸馏）。
````

## 样例5

````markdown
同学你好，这个问题是理解Transformer的“任督二脉”。我们不用数学堆砌，一步步把直觉和细节讲透。

---

### 1. 直观类比：Q、K、V 分别代表什么？

想象你在**图书馆找书**：

- **Query（查询）** = 你脑子里的**需求**。比如“我想找关于深度学习的入门书”。
- **Key（键）** = 每本书的**标签/索引**。比如书脊上写着“AI”、“Python”、“数学”、“进阶”等。
- **Value（值）** = 那本书的**实际内容**（文字、知识）。

**过程**：你把你的需求（Q）和所有书的标签（K）做匹配，找出最相关的几本（高分标签），然后你主要去阅读这几本书的内容（V）。  
注意：**Key 和 Value 可以分离**——标签是简短的，内容是丰富的，它们只是同一本书的不同表示。

---

### 2. 缩放点积注意力公式

\[
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V
\]

---

### 3. 公式中每一项的含义（包括 \(d_k\)）

- **\(QK^T\)**（点积）  
  计算 Query 和每个 Key 的**相似度得分**。点积越大，表示两者的方向越接近，相关性越高。

- **\(\sqrt{d_k}\)**（缩放因子）  
  \(d_k\) 是 **Key（也是 Query）的维度**。  
  为什么要除？当维度 \(d_k\) 较大时，点积的值会变得很大，导致 softmax 输出的概率分布**非常尖锐**（几乎独热），梯度极小，不利于训练。除以 \(\sqrt{d_k}\) 可以把方差拉回 1，保持梯度平稳。

- **softmax**  
  把原始得分转换成**概率权重**（所有位置权重之和为 1）。它决定了我们“关注”每个 Value 的程度。

- **乘以 V**  
  用 softmax 输出的权重，对所有的 Value 做**加权求和**。最终输出是融合了上下文信息的向量。

---

### 4. 简短例子：根据 Query 找 Key，再汇总 Value

假设句子是 **“我 喜欢 猫”**，我们计算“喜欢”这个词的新表示（Query 来自“喜欢”）。

- **Q（“喜欢”）** = [0.8, 0.1]  
- **K（“我”）** = [0.9, 0.0] → 点积 = 0.72  
- **K（“喜欢”）** = [0.7, 0.3] → 点积 = 0.59  
- **K（“猫”）** = [0.1, 0.9] → 点积 = 0.17  

经过 softmax（假设温度缩放后）：权重约为 **0.45（我），0.35（喜欢），0.20（猫）**。  
然后 **V（“我”）** 的内容向量 ×0.45 + **V（“喜欢”）** ×0.35 + **V（“猫”）** ×0.20。  
结果：新的“喜欢”表示中，融入了 45% 的“我”信息、35% 自身信息和 20% 的“猫”信息 —— 这就完成了上下文感知。

---

### 5. 初学者容易混淆的地方（重点）

| 混淆点                                                     | 正确理解                                                                                                           |
| ---------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| **Q、K、V 不是固定的词向量**                               | 它们都是**输入 X 乘以不同的权重矩阵**（\(W_Q, W_K, W_V\)）得到的。同一个词在不同注意力头中，Q/K/V 都不同。         |
| **Key 和 Value 总是一一对应同一个词**                      | 是的，每个位置的 Key 和 Value 来自同一个输入词，但**作用不同**：Key 用来被匹配，Value 用来提供内容。               |
| **Q 和 K 必须维度相同，V 可以不同**                        | 公式要求 \(Q\) 和 \(K^T\) 能相乘，所以维度必须一致（都是 \(d_k\)）。而 V 的维度 \(d_v\) 可以独立设定（输出维度）。 |
| **除以 \(\sqrt{d_k}\) 是为了防止梯度消失，不是防止过拟合** | 很多人误以为是归一化，其实是**方差控制**，保证 softmax 输入不会过大。                                              |
| **Attention 权重是“软”的**                                 | 它不是只选一个最相关的 Key（硬注意），而是对所有 Key 都分配权重，只是权重大小不同。                                |

---

如果还有疑问，比如“为什么叫缩放点积”或“多头注意力怎么用”，我们下次再聊。先掌握这个核心，后面的都是它的变体。加油！🧠
````

## 样例6

````markdown
同学们好，我是你们的课程助教。今天我们把多分类中最核心的“Softmax + 交叉熵”组合彻底讲透。请跟着下面的结构一步步来。

---

### 1. Softmax 的输入和输出分别是什么

**输入**：一个任意实数值的向量 **z** = [z₁, z₂, ..., z_K]，长度等于类别数 K。  
这个向量通常来自神经网络最后一层（全连接层）的输出，称为**logits**（逻辑值）。它没有概率含义，可以是正数、负数或零，大小关系代表模型对各类别的“原始打分”。

**输出**：一个与输入同维度的概率向量 **p** = [p₁, p₂, ..., p_K]，满足：
- 每个 pᵢ 在 (0, 1) 之间；
- 所有 pᵢ 之和等于 1。

计算公式（对第 i 类）：
$$
p_i = \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}}
$$
符号含义：  
- e 是自然常数（约2.718），取指数是为了让得分都变成正数，且拉开差距；  
- 分母是所有类别的指数和，作用是归一化，保证总和为1。

**直观理解**：Softmax 把“原始分数”转化为“模型认为每个类别的概率”，分数越高，对应概率越大。

---

### 2. 交叉熵损失衡量的是什么

交叉熵（Cross-Entropy）衡量**两个概率分布之间的差异**。  
在多分类中，这两个分布是：
- **真实分布** y（one-hot 编码）：例如第 k 类为1，其余为0；
- **预测分布** p（Softmax 输出的概率）。

对于单个样本，损失公式为：
$$
L = -\sum_{i=1}^{K} y_i \log(p_i)
$$
符号含义：  
- yᵢ 是真实标签的第 i 个分量（0 或 1）；  
- pᵢ 是模型预测的第 i 类概率；  
- log 是自然对数。

由于 y 是 one-hot，只有真实类别 y_c = 1，其余为0，所以公式简化为：
$$
L = -\log(p_c)
$$
其中 p_c 是模型分配给**真实类别**的概率。

**衡量的是什么**：它衡量的是“为了表示真实标签，预测分布所需要付出的信息量（以比特为单位）”。  
- 当 p_c 接近 1 时，损失接近 0（模型很确定且正确）；  
- 当 p_c 接近 0 时，损失趋向无穷大（模型极不看好真实类别）。

---

### 3. 为什么二者常被组合用于多分类

二者是天作之合，原因有三：

1. **概率要求**：交叉熵要求输入是概率分布（和为1，非负），而 Softmax 恰好将 logits 转化为合法的概率分布。

2. **梯度优良**：将 Softmax 和交叉熵联合求导，得到的梯度形式非常简洁：
   $$
   \frac{\partial L}{\partial z_i} = p_i - y_i
   $$
   这意味着：梯度 = 预测概率 − 真实标签。  
   - 如果预测偏高（pᵢ > yᵢ），梯度为正，更新时降低该 logit；  
   - 如果预测偏低（pᵢ < yᵢ），梯度为负，更新时提高该 logit。  
   这个形式使得反向传播非常稳定，不会出现梯度消失或爆炸（相比单独用均方误差）。

3. **优化目标明确**：最小化交叉熵等价于最大化真实类别的对数似然，即鼓励模型把概率质量集中在正确的类别上。

---

### 4. 三分类数值例子：预测概率和真实标签如何产生损失

假设三分类问题：类别 A、B、C。  
一个样本的真实标签是 **B**，所以 one-hot 向量为 **y** = [0, 1, 0]。

神经网络输出的 logits 为 **z** = [2.0, 1.0, 0.1]。

**Step 1: Softmax 计算概率**  
指数值：  
e² ≈ 7.39, e¹ ≈ 2.72, e⁰·¹ ≈ 1.11  
分母 = 7.39 + 2.72 + 1.11 = 11.22  

概率：  
p_A = 7.39/11.22 ≈ 0.659  
p_B = 2.72/11.22 ≈ 0.242  
p_C = 1.11/11.22 ≈ 0.099  

预测分布 **p** = [0.659, 0.242, 0.099]（模型认为最可能是 A，但真实是 B）。

**Step 2: 计算交叉熵损失**  
因为 y_B = 1，其他为0：
$$
L = -\log(p_B) = -\log(0.242) ≈ 1.418
$$
这个损失值 > 0。如果模型预测 p_B = 0.99，则损失 ≈ 0.01，很小；如果 p_B = 0.001，损失 ≈ 6.9，很大。

---

### 5. 为什么“置信度高但预测错误”时损失会很大

假设另一个场景：模型对同一个真实标签 B 的样本，输出了 logits = [10.0, 0.0, 0.0]。  
Softmax 后：  
e¹⁰ ≈ 22026, e⁰=1, e⁰=1 → 分母 ≈ 22028  
p_A ≈ 0.9999, p_B ≈ 0.000045, p_C ≈ 0.000045  

此时模型 **以极高置信度（99.99%）预测为 A**，但真实是 B。

交叉熵损失：
$$
L = -\log(0.000045) ≈ 10.0
$$
这非常大（远大于例1的1.418）。

**直观原因**：  
交叉熵取 log 的特性是**惩罚极低概率**。当模型把真实类别的概率压到接近 0 时，-log(p) 会急剧增长（因为 log(0) → −∞）。  
这意味着：**错误且自信**比**错误但犹豫**要糟糕得多——因为模型不仅错了，还固执地否定了正确答案，这在实际应用（如医疗、自动驾驶）中是危险的。

**教学要点**：这个性质迫使模型在训练中既要做对，又要保持概率分配的适度保守，避免过度自信地犯错。
````

````markdown
**1. 主要错误或遗漏**

**核心错误：缺少 `optimizer.zero_grad()`**。

在每次迭代中，反向传播（`loss.backward()`）计算的梯度会**累加**到参数的 `grad` 属性上，但代码中从未在更新前将历史梯度清零。

**2. 为什么该问题会影响训练**

-   **梯度累积叠加**：如果没有清零，每一步的梯度都会与上一步的梯度相加。这会导致参数更新的方向严重偏离正确的梯度方向，梯度值会持续膨胀。
-   **模型无法收敛**：由于更新方向错误且步长（学习率 × 累积梯度）失控，损失值通常会剧烈震荡甚至发散（变为 NaN），导致模型完全无法学习有效特征。

---

**3. 修改后的训练循环代码（必须修改部分）**

```python
for epoch in range(5):
    for x, y in train_loader:
        # 【必须】在反向传播前清空上一步的梯度
        optimizer.zero_grad()
        
        pred = model(x)
        loss = criterion(pred, y)
        loss.backward()
        optimizer.step()
```

---

**4. 必须修改与可选改进建议**

-   **必须修改（影响正确性）**：
    -   添加 `optimizer.zero_grad()`，且位置必须在 `loss.backward()` 之前。通常紧贴在 `pred` 计算之前或 `loss` 计算之前。
````

````markdown
### 深度学习考前复习提纲（基于课堂笔记）

---

#### 一、优化方法

- **SGD（随机梯度下降）**：每次仅用一个或一小批样本计算梯度来更新参数，计算开销小，但梯度噪声大，收敛路径易震荡。  
- **Momentum（动量法）**：在SGD基础上累加历史梯度方向（动量项），能加速收敛并抑制震荡，尤其适用于狭窄峡谷或局部极值区域。  
- **Adam（自适应矩估计）**：结合动量（一阶矩）和RMSprop的自适应学习率（二阶矩），为每个参数动态调整步长，适合稀疏梯度或非平稳目标。

---

#### 二、正则化方法

- **L2 正则化（权重衰减）**：在损失函数中添加权重平方和惩罚项，迫使权重趋向于较小值，从而限制模型复杂度、缓解过拟合。  
- **Dropout**：训练时按一定概率随机“丢弃”神经元（即暂时不参与前向和反向传播），迫使网络学习冗余表示，相当于隐式集成多个子网络。  
- **数据增强**：对训练数据施加随机变换（如旋转、裁剪、翻转等）生成新样本，增加数据多样性，降低对特定样本特征的依赖。  
- **Early Stopping**：在验证集性能不再提升时提前终止训练，防止训练后期过拟合，是一种代价低廉的正则化手段。

---

#### 三、学习率影响

- **学习率过大**：参数更新步长太大，可能导致损失震荡甚至发散，无法收敛到良好极小值。  
- **学习率过小**：收敛速度极慢，且可能陷入局部极小值或鞍点区域长期停滞。  
- **学习率与优化器配合**：Adam等自适应方法对初始学习率相对不敏感，但SGD/Momentum对学习率非常敏感，需要更精细的调参或学习率衰减策略。

---

#### 四、易混淆概念（对比澄清）

- **Momentum vs. Adam**：两者都含“动量”思想，但Adam额外有自适应逐参数学习率；Momentum所有参数共用同一学习率，而Adam每个参数有独立步长。  
- **L2 正则化 vs. 权重衰减**：在标准SGD下等价，但在自适应优化器（如Adam）中二者实现细节不同（Adam中L2不等于简单权重衰减）。  
- **Dropout（训练）vs. Dropout（测试）**：训练时随机丢弃并缩放保留神经元（通常除以保留概率），测试时不丢弃且所有神经元参与，但输出要乘以保留概率（或训练时已缩放）。  
- **Early Stopping vs. 正则化**：Early Stopping不显式修改损失函数或网络结构，而是通过限制优化迭代次数间接限制模型容量，常被视为隐式正则化。  
- **数据增强 vs. 其他正则化**：数据增强作用于输入空间，增加有效训练集规模；L2/Dropout作用于模型参数或结构，两者可互补并用。

---

#### 五、可能的考试提问角度（≥5个）

4. **给定训练损失持续下降但验证损失先降后升，你应该优先调整哪个超参数？并说明理由。**  
5. **学习率从0.1改为0.001后，SGD和Adam的收敛行为分别会发生怎样的变化？哪个更稳定？**
````

````markdown
### 3. 关键概念解释

- **Transformer**：最初用于机器翻译的神经网络结构，核心是“自注意力”层。它不像RNN那样逐个处理词，而是并行处理整个序列，擅长捕捉全局关系。这里被移植到图像上。

- **Self-attention（自注意力）**：一种计算“序列中每个元素与其他所有元素相关性”的机制。例如，对每个图像块，它会计算该块与所有其他块的相似度得分，然后根据得分加权融合信息。这样每个位置都能“看到”全局。

- **Convolutional inductive bias（卷积归纳偏置）**：指CNN自带的两个先验假设——**局部性**（相邻像素相关）和**平移等变性**（物体出现在不同位置，卷积核都能识别）。这些是先验知识，不是从数据学来的。本文去掉了它们，让模型从数据中自己学习空间关系。

- **Long-range dependencies（长程依赖关系）**：指图像中距离很远的两个区域之间的语义关联。例如，分类“人”时，头部和脚部虽然相隔很远，但都需要被考虑。CNN通过堆叠很多层才能覆盖远距离，而自注意力一步到位。

- **Image patches（图像块）**：为了把图像输入Transformer（它吃序列），作者把一张图切成固定大小的小方块（如16×16像素），每个小方块展平为一个向量，相当于“序列中的一个词”。这样图像就变成了一个“词序列”。
````
